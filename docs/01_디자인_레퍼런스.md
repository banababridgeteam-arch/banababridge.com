## 바나바 적용 원칙 (회의 합의 반영)

1. **최적화 80 / 디자인 20** — Awwwards식 풀 3D·WebGL은 금지. 캔버스 2D·CSS 모션까지만.
2. 첫 화면 3초 안에 "무슨 회사인지" 읽혀야 함. 설명이 필요한 홈페이지는 실패.
3. 레퍼런스 카드가 늘어나는 구조가 최우선 (지금 4건 → 계속 추가).
4. 폰트는 Pretendard 하나로 통일 (동적 서브셋 = 한글 웹폰트 성능 최적).
5. 색은 다크(#0a0a0e) + 샴페인 골드(#cfa96b) 2색 체계 유지 — 고급감은 색 수가 적을수록 올라감.
